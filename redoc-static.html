  • Helpful/not helpful votes
  • Ad reports
  • Not ad reports

required

userId
required
string (Userid)

string (Feedbacktype)

Trigger Model Training

Trigger model training.

Trains a new model using all available admin labels (gold labels). The training is synchronous (MVP), but could be made async with Celery.

Args: request: Training request db: Database session admin_key: Verified admin API key

Returns: Training response with job acceptance status

Raises: 401: Invalid admin API key 409: Insufficient training data 422: Invalid training parameters 500: Training failed

header Parameters
X-Admin-Key
required
string (X-Admin-Key)
Request Body schema: application/json
required
adminId
required
string (Adminid)

Admin user ID requesting training

mode
string (Mode)
Default: "full"

Training mode: 'full' or 'incremental'

minNewLabels
integer (Minnewlabels) >= 1
Default: 20

Minimum number of new labels required to proceed

activateIfBetter
boolean (Activateifbetter)
Default: true

Whether to activate model if it performs better

Responses

/api/admin/model/train

Request samples

  • Payload
Content type
application/json
{
  • "adminId": "string",
  • "mode": "full",
  • "minNewLabels": 20,
  • "activateIfBetter": true
}

Response samples

  • 200
  • 422
Content type
application/json
{
  • "requestId": "string",
  • "accepted": true,
  • "jobId": "string",
  • "message": "string"
}
